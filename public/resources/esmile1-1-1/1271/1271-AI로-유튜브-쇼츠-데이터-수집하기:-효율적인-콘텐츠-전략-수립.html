AI로 유튜브 쇼츠 데이터 수집하기: 효율적인 콘텐츠 전략 수립

유튜브 쇼츠가 급부상하면서 많은 크리에이터들이 이 플랫폼에 주목하고 있습니다. 하지만 효과적인 쇼츠 전략을 수립하기 위해서는 데이터 분석이 필수적입니다. 이번 포스트에서는 AI를 활용해 유튜브 쇼츠 데이터를 수집하는 방법을 상세히 알아보겠습니다.

왜 쇼츠 데이터 수집이 중요한가?

쇼츠 데이터를 수집하면 다음과 같은 이점이 있습니다:

  1. 트렌드 파악: 어떤 주제의 쇼츠가 인기 있는지 알 수 있습니다.
  2. 경쟁 분석: 다른 크리에이터들의 전략을 벤치마킹할 수 있습니다.
  3. 콘텐츠 최적화: 조회수, 좋아요 수 등을 분석해 더 나은 콘텐츠를 만들 수 있습니다.
  4. 시간 절약: 수동으로 데이터를 수집하는 것보다 훨씬 효율적입니다.

이제 실제로 AI를 활용해 쇼츠 데이터를 수집하는 방법을 단계별로 알아보겠습니다.

function fetchYouTubeShorts(query, maxResults) {
  var results = YouTube.Search.list('snippet', {
    q: query,
    maxResults: maxResults,
    type: 'video',
    videoDuration: 'short'
  });
  return results.items;
}

세 번째 방법: 영상 비율로 쇼츠 찾기

세 번째 방법은 영상의 비율을 기준으로 쇼츠를 찾는 방식입니다. 쇼츠는 일반적으로 9:16 비율을 가지고 있기 때문에, 이 비율을 이용해 쇼츠를 식별할 수 있습니다.

코드 수정

searchShorts 함수를 다음과 같이 수정합니다:

function searchShorts(query, maxResults, publishedAfter) {
  var results = YouTube.Search.list('snippet', {
    q: query,
    maxResults: maxResults,
    type: 'video',
    publishedAfter: publishedAfter,
    videoDefinition: 'high'
  });

  var shortsData = [];

  for (var i = 0; i < results.items.length; i++) {
    var videoId = results.items[i].id.videoId;
    var videoDetails = YouTube.Videos.list('snippet,contentDetails', {id: videoId});

    if (videoDetails.items && videoDetails.items.length > 0) {
      var width = videoDetails.items[0].snippet.thumbnails.default.width;
      var height = videoDetails.items[0].snippet.thumbnails.default.height;

      // 9:16 비율 확인 (허용 오차 ±5%)
      if (Math.abs((height / width) - (16 / 9)) < 0.05) {
        shortsData.push(parseVideoData(videoDetails.items[0], results.items[i]));
      }
    }
  }

  return shortsData;
}

이 코드는 각 비디오의 썸네일 비율을 확인하여 9:16 비율(±5% 오차 허용)에 해당하는 영상만 쇼츠로 간주합니다.

결과 분석

이 방법을 사용하면 다음과 같은 장단점이 있습니다:

장점:

  • 정확도가 높습니다. 실제 쇼츠 형식의 영상을 더 정확하게 찾을 수 있습니다.
  • URL에 "shorts"가 포함되지 않은 쇼츠도 찾을 수 있습니다.

단점:

  • API 호출 횟수가 증가하여 할당량을 더 빨리 소진할 수 있습니다.
  • 처리 시간이 더 오래 걸릴 수 있습니다.

추가 개선 사항

  1. 썸네일 미리보기: 썸네일을 클릭하면 영상을 바로 볼 수 있도록 하이퍼링크를 추가합니다.
  2. 데이터 필터링: 조회수, 좋아요 수, 댓글 수 등으로 데이터를 필터링할 수 있는 기능을 추가합니다.
  3. 주기적 업데이트: 매일 또는 매주 자동으로 데이터를 업데이트하는 트리거를 설정합니다.
  4. 차트 생성: 수집된 데이터를 바탕으로 트렌드를 시각화하는 차트를 만듭니다.
  5. 다국어 지원: 검색어를 다양한 언어로 입력할 수 있도록 지원합니다.

세 가지 방법 중 두 번째 방법(URL에 "shorts"가 포함된 영상 찾기)이 가장 효율적이고 정확한 것으로 보입니다. 하지만 프로젝트의 요구사항과 사용 목적에 따라 다른 방법을 선택하거나 여러 방법을 조합하여 사용할 수 있습니다.
