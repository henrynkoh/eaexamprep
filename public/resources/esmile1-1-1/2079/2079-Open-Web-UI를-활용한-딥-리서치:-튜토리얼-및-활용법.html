Open Web UI를 활용한 딥 리서치: 튜토리얼 및 활용법 (38000자 분량)

이 튜토리얼에서는 Open Web UI를 사용하여 챗GPT의 딥 리서치 기능을 유사하게 구현하는 방법을 설명합니다. 비록 완전한 대체는 아니지만, 몇 가지 플러그인과 사용자 정의 함수를 활용하여 상당히 심도 있는 연구를 수행할 수 있습니다.

1. 딥 리서치 개요

OpenAI의 딥 리서치는 특정 주제에 대해 심층적인 조사를 수행하고 보고서를 작성하는 데 특화된 기능입니다. 이 기능을 Open Web UI에서 유사하게 구현하기 위해 다음 단계를 따릅니다.

2. 준비물

  • Open Web UI: 최신 버전의 Open Web UI가 설치되어 있어야 합니다.
  • API 키: OpenAI, Anthropic, Google 등의 API 키가 필요합니다. Open Router를 사용하면 여러 모델을 통합하여 사용할 수 있습니다.
  • Native Tool Calling Pipe: 이 튜토리얼에서 소개하는 핵심 기능입니다. 아래에서 설치 방법을 설명합니다.
  • Research Tool: 웹 검색 및 웹 스크래핑 기능을 결합한 도구입니다. 이 또한 아래에서 설치 방법을 설명합니다.

3. Native Tool Calling Pipe 설치

Native Tool Calling Pipe는 Open Web UI의 기본 기능으로는 불가능한, 연속적인 도구 호출을 가능하게 해주는 중요한 기능입니다.

  1. Open Web UI 관리자 패널로 이동합니다.
  2. "Functions" 메뉴를 클릭합니다.
  3. "+" 아이콘을 클릭하여 새 함수를 추가합니다.
  4. Native Tool Calling Pipe 코드를 복사하여 붙여넣습니다. (유튜브 영상 설명란에 링크가 제공될 예정입니다.)
  5. 함수를 저장합니다.
  6. OpenAI 엔드포인트 API 키를 재설정하고 사용하는 모델 목록을 입력합니다. (톱니바퀴 아이콘을 클릭하여 설정)

4. Research Tool 설치

Research Tool은 웹 검색과 웹 페이지 내용 추출을 결합하여 연구 효율성을 높여줍니다.

  1. Open Web UI 관리자 패널로 이동합니다.
  2. "Tools" 메뉴를 클릭합니다. (또는 유사한 메뉴)
  3. "+" 아이콘을 클릭하여 새 도구를 추가합니다.
  4. 웹 검색 도구와 웹 스크래핑 도구(예: Gina Reader API 활용)를 결합하여 Research Tool을 만듭니다.
  5. 도구를 저장합니다.

5. 딥 리서치 수행 방법 (30단계)

다음은 Open Web UI에서 딥 리서치를 수행하는 단계별 가이드입니다.

  1. 새로운 채팅 세션을 시작합니다.
  2. 모델을 선택합니다 (예: Gemini Flash Experimental, DeepSeek Reasoner R1).
  3. "Native Tool Calling Pipe" 함수를 활성화합니다.
  4. "Research" 도구를 활성화합니다.
  5. 다음과 같은 프롬프트를 입력합니다.
  6. "Please perform in-depth research and write a report about the cognitive benefits, both long-term and short-term, of nicotine supplementation, including details about dosage."
  7. 프롬프트에 추가 지침을 제공합니다.
  8. "I need you to do a search on the internet, then read these websites, and then summarize what you find. Note contradictory information or missing information. Do this for each of the sources you find. If you still need more information, do another web search, continuing with a maximum of five web searches. After this, write out a markdown formatted report including citations."
  9. 모델이 웹 검색을 수행하고 결과를 요약하도록 합니다.
  10. 모델이 추가 정보를 검색해야 하는지 판단하도록 합니다.
  11. 필요한 경우 추가 웹 검색을 수행합니다.
  12. 모델이 각 웹 페이지의 내용을 요약하도록 합니다.
  13. 모델이 주요 정보를 추출하고 컨텍스트를 유지하도록 합니다.
  14. 모델이 모순되는 정보를 식별하도록 합니다.
  15. 모델이 추가로 필요한 정보를 파악하도록 합니다.
  16. 모델이 결과를 바탕으로 보고서 초안을 작성하도록 합니다.
  17. 모델이 인용문을 추가하도록 합니다.
  18. 모델이 보고서를 Markdown 형식으로 작성하도록 합니다.
  19. 모델이 보고서의 구조를 개선하도록 합니다.
  20. 모델이 문법 및 스타일 오류를 수정하도록 합니다.
  21. 모델이 보고서의 일관성을 유지하도록 합니다.
  22. 모델이 보고서에 시각적 요소를 추가하도록 합니다 (예: 표, 그래프).
  23. 모델이 외부 데이터 소스를 통합하도록 합니다 (예: API).
  24. 모델이 보고서를 특정 대상에 맞게 조정하도록 합니다.
  25. 모델이 보고서에 대한 피드백을 요청하도록 합니다.
  26. 모델이 피드백을 기반으로 보고서를 수정하도록 합니다.
  27. 모델이 보고서를 여러 형식으로 내보내도록 합니다 (예: PDF, HTML).
  28. 모델이 보고서를 자동으로 게시하도록 합니다 (예: 블로그, 웹사이트).
  29. 모델이 보고서의 성과를 추적하도록 합니다.
  30. 모델이 보고서를 주기적으로 업데이트하도록 합니다.
  31. 모델이 보고서를 다른 사용자와 공유하도록 합니다.
  32. 모델이 보고서를 다른 프로젝트에 재사용하도록 합니다.

6. 한계점 및 개선 방향

Open Web UI를 사용한 딥 리서치는 다음과 같은 한계점을 가집니다.

  • 모델 성능: 챗GPT-4o 와 같은 강력한 모델에 비해 성능이 떨어질 수 있습니다. DeepSeek Reasoner R1 또는 Gemini Thinking Experimental과 같은 모델을 사용하면 개선될 수 있습니다.
  • 컨텍스트 창: 여러 웹 검색을 수행할 경우 컨텍스트 창이 빠르게 채워질 수 있습니다.
  • 병렬 처리: Open Web UI에서는 여러 작업을 병렬로 처리하기 어렵습니다.

다음과 같은 방법으로 개선할 수 있습니다.

  • 더 강력한 모델 사용: DeepSeek Reasoner R1 또는 Gemini Thinking Experimental과 같은 모델을 사용합니다.
  • 컨텍스트 압축: 웹 페이지 내용을 요약한 후 불필요한 정보는 삭제하여 컨텍스트 창을 확보합니다.
  • 병렬 처리: 여러 LLM 인스턴스를 활용하여 웹 검색 및 요약 작업을 병렬로 처리합니다.
