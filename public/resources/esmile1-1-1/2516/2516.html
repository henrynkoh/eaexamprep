 "Smithey GitHub API 도구" 사용 방법 및 상세 설명
안녕하세요! 오늘은 GitHub API를 활용하여 파일 작업, 저장소 관리, 검색 등의 기능을 제공하는 Smithey라는 도구에 대해 자세히 알아보겠습니다. 이 도구는 GitHub API를 기반으로 다양한 작업을 자동화할 수 있는 강력한 기능을 제공합니다. 아래는 Smithey 도구의 주요 기능과 사용 방법을 한글로 상세히 정리한 내용입니다. 
Smithey란 무엇인가요?
Smithey는 GitHub API를 활용하여 파일 생성, 저장소 검색, 저장소 생성, 파일 내용 조회 등 다양한 작업을 수행할 수 있는 도구입니다. Smithey는 GitHub 계정과 연동하여 사용자가 GitHub에서 수행할 수 있는 작업을 더 쉽게 관리할 수 있도록 설계되었습니다. 이 도구는 특히 개발자나 GitHub를 자주 사용하는 사용자에게 유용하며, API 호출을 통해 작업을 자동화할 수 있는 점이 큰 장점입니다.
Smithey의 주요 기능
Smithey는 다음과 같은 주요 기능을 제공합니다:
  • create_or_update_file: GitHub 저장소에서 파일을 생성하거나 업데이트합니다.
  • search_repositories: GitHub 저장소를 검색합니다.
  • create_repository: 새로운 GitHub 저장소를 생성합니다.
  • get_file_contents: GitHub 저장소에서 파일이나 디렉토리의 내용을 가져옵니다.
이 외에도 13개 이상의 추가 도구가 있으며, 이를 통해 GitHub API를 활용한 다양한 작업을 수행할 수 있습니다.
Smithey 사용 방법: 30단계로 세분화된 가이드
Smithey를 효과적으로 사용하기 위해 아래 30단계로 나누어 상세히 설명하겠습니다. 이 가이드는 초보자도 쉽게 따라 할 수 있도록 구성되었습니다.
1단계: Smithey 웹사이트 접속
Smithey를 사용하려면 먼저 공식 웹사이트에 접속해야 합니다. 브라우저에서 smithey.ai로 이동하세요.
2단계: GitHub 계정 확인
Smithey는 GitHub API를 사용하므로, GitHub 계정이 필요합니다. GitHub 계정이 없다면 먼저 계정을 생성하세요.
3단계: Smithey 로그인
Smithey 웹사이트에서 로그인 버튼을 클릭하여 GitHub 계정으로 로그인합니다. GitHub OAuth를 통해 인증이 진행됩니다.
4단계: GitHub API 토큰 생성
Smithey는 GitHub API를 사용하기 위해 개인 액세스 토큰(Personal Access Token)이 필요합니다. GitHub 계정 설정(Settings)으로 이동하세요.
5단계: 개발자 설정으로 이동
GitHub 계정 설정에서 "Developer settings" 메뉴를 선택합니다.
6단계: 개인 액세스 토큰 생성
"Personal access tokens" 섹션에서 "Generate new token" 버튼을 클릭합니다.
7단계: 토큰 이름 입력
토큰 이름을 입력합니다. 예를 들어, "Smithey Token"이라고 이름을 지정할 수 있습니다.
8단계: 권한 설정
Smithey가 필요한 권한을 선택합니다. 최소한 repo (저장소 접근) 권한이 필요합니다. 추가적으로 admin:repo_hook이나 user 권한도 유용할 수 있습니다.
9단계: 토큰 생성 및 복사
권한 설정 후 "Generate token" 버튼을 클릭하여 토큰을 생성하고, 생성된 토큰을 복사합니다. 이 토큰은 한 번만 표시되므로 반드시 저장하세요.
10단계: Smithey로 돌아가기
Smithey 웹사이트로 돌아가서 "Configuration" 섹션으로 이동합니다.
11단계: GitHub Personal Access Token 입력
"Configuration" 섹션에서 githubPersonalAccessToken 입력란에 복사한 토큰을 붙여넣습니다.
12단계: Connect 버튼 클릭
토큰 입력 후 "Connect" 버튼을 클릭하여 Smithey와 GitHub 계정을 연동합니다.
13단계: 연동 확인
연동이 성공하면 Smithey 대시보드에서 GitHub 계정(
@smithey
-ai/github)이 활성화된 것을 확인할 수 있습니다.
14단계: Overview 탭 확인
Smithey 대시보드에서 "Overview" 탭을 클릭하여 주요 기능을 확인합니다.
15단계: Tools 탭으로 이동
"Tools" 탭을 클릭하여 Smithey가 제공하는 도구 목록을 확인합니다.
16단계: create_or_update_file 도구 선택
파일 생성/업데이트를 위해 "create_or_update_file" 도구를 선택합니다.
17단계: 저장소 이름 입력
파일을 생성하거나 업데이트할 GitHub 저장소 이름을 입력합니다. 예: my-repo.
18단계: 파일 경로 입력
파일 경로를 입력합니다. 예: path/to/file.txt.
19단계: 파일 내용 입력
업데이트할 파일 내용을 입력합니다. 예: "Hello, this is a test file."
20단계: 커밋 메시지 작성
파일 변경에 대한 커밋 메시지를 작성합니다. 예: "Add test file".
21단계: 실행 버튼 클릭
모든 정보를 입력한 후 "Execute" 또는 "Run" 버튼을 클릭하여 파일 생성/업데이트 작업을 실행합니다.
22단계: 작업 결과 확인
작업이 완료되면 결과 메시지를 확인합니다. 성공적으로 파일이 생성/업데이트되었는지 확인하세요.
23단계: search_repositories 도구 선택
저장소를 검색하려면 "search_repositories" 도구를 선택합니다.
24단계: 검색 키워드 입력
검색할 키워드를 입력합니다. 예: "machine learning".
25단계: 검색 실행
검색 버튼을 클릭하여 GitHub 저장소를 검색합니다.
26단계: 검색 결과 확인
검색 결과로 반환된 저장소 목록을 확인하고, 필요한 저장소를 선택합니다.
27단계: create_repository 도구 선택
새로운 저장소를 생성하려면 "create_repository" 도구를 선택합니다.
28단계: 저장소 이름 및 설정 입력
새 저장소 이름을 입력하고, 공개/비공개 여부 등 설정을 선택합니다.
29단계: 저장소 생성 실행
"Create" 버튼을 클릭하여 새로운 저장소를 생성합니다.
30단계: 생성된 저장소 확인
GitHub 계정에서 새로 생성된 저장소가 추가되었는지 확인합니다.
Smithey의 추가 정보
  • 월간 API 호출 수: Smithey는 월간 약 102,914회의 API 호출을 처리합니다.
  • 로컬 여부: 이 도구는 로컬에서 실행되지 않으며, 클라우드 기반으로 작동합니다.
  • 출시일: Smithey는 2025년 1월 18일에 공개되었습니다.
Smithey는 GitHub API를 활용한 다양한 작업을 자동화할 수 있는 강력한 도구로, 개발자 및 GitHub 사용자에게 큰 도움이 될 것입니다.
관련 프롬프트
Smithey 대시보드에서 "Search or prompt for servers..."라는 프롬프트 입력란이 있습니다. 이 입력란을 활용하여 다음과 같은 프롬프트를 사용할 수 있습니다:
  • "Create a new repository named 'my-project' with a README file."
  • "Search for repositories related to 'artificial intelligence'."
  • "Update the file 'README.md' in the repository 'my-repo' with the content 'Welcome to my project!'."
이 프롬프트들은 Smithey의 기능을 더 쉽게 활용할 수 있도록 도와줍니다.
