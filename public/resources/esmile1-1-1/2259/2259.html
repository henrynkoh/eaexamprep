세금 신고의 정확성

세금 신고의 정확성은 매우 중요합니다. 납세자는 세금 신고 방법에 관계없이 신고된 금액에 대해 항상 책임을 집니다1. 세무사로서 고객을 위해 최선을 다해야 하지만, IRS는 궁극적으로 납세자가 세금 신고서에 기재된 내용에 대해 책임이 있다고 봅니다1.

세금 소프트웨어는 매우 유용한 도구이지만, 전적으로 의존해서는 안 됩니다1. 단순히 숫자를 입력하면 자동으로 정확한 세금 신고서가 나온다고 가정해서는 안 됩니다1. 세금에 대해 이해하지 못하는 사람이 세금 신고를 할 수 없는 이유가 바로 이 때문입니다1.

 

소프트웨어가 실수를 할 수도 있고, 더 자주 발생하는 경우는 데이터를 입력하는 사람이 실수를 하는 것입니다1. 금액을 잘못 입력하거나, 잘못된 위치에 입력하거나, 한 곳 대신 두 곳에 입력하여 중복 입력이 되는 등의 실수가 있을 수 있습니다1.

 

세금 법원에서는 "소프트웨어 때문이다"라는 변명이 통하지 않습니다1. 컴퓨터와 관련해서는 "쓰레기를 넣으면 쓰레기가 나온다(Garbage In, Garbage Out)"라는 개념이 적용됩니다1.

일반적인 실수 사례

실수는 종종 중복 입력으로 인해 발생합니다1. 소프트웨어는 다른 부분에서 사용할 정보를 자동으로 가져오려고 하지만, 세무사가 이를 인식하지 못하고 같은 정보를 다시 입력하면 이중 계산이 될 수 있습니다1.

 

일반적인 실수 사례는 다음과 같습니다:

  1. HSA(Health Savings Account) 기여금
  2. 주 세금, 특히 주 소득세
  3. 부동산세
  4. Schedule C 소득, 특히 1099 양식 관련

예를 들어, HSA 기여금의 경우 W-2에 이미 포함되어 있는데 HSA 기여금 화면에 다시 입력하면 두 번 계산될 수 있습니다1.

실수 발견 시 대처 방법

이미 신고한 후 IRS가 지적하기 전에 오류를 발견한 경우, 수정된 신고서를 제출해야 합니다1. IRS가 오류를 발견한 경우, 통지를 보내고 신고서를 조정한 후 추가 납부 또는 환급 금액을 알려줄 것입니다1. 때로는 IRS가 오류가 전체 신고서에 어떤 영향을 미치는지 정확히 알지 못할 수 있으므로, 수정된 신고서 제출을 포함한 다른 조치를 취하도록 지시할 수 있습니다1.

기록 유지

기록 보관

세금 신고서 작성 시 고객으로부터 많은 문서를 요청하고 받게 됩니다1. 이전 연도 세금 신고서, W-2, 내부 회계 기록, 은행 명세서, 비용 영수증 등을 받을 수 있습니다1. 이러한 모든 문서의 사본(종이 또는 스캔본)을 보관해야 합니다1.

 

고객 미팅 노트와 이메일도 보관해야 합니다1. 고객과 미팅 후 내용을 요약하여 이메일로 보내고 확인받는 것이 좋습니다1. 단, 법적으로 민감한 사항이나 범죄와 관련된 내용은 이메일에 포함하지 않도록 주의해야 합니다1.

기록 보관 기간

기록은 일반적으로 해당 신고서에 대한 소멸시효가 유효한 동안 보관해야 합니다1. 보통 신고서 제출일 또는 마감일 중 늦은 날짜로부터 3년입니다1.

 

부동산 거래의 경우 더 오랜 기간 동안 기록을 보관해야 할 수 있습니다1. 예를 들어, 기업이 건물을 구매하고 20년 후에 판매한 경우, 원래 매매 계약서와 구매 가격, 감가상각 등을 증명하는 모든 문서를 해당 부동산을 처분한 연도의 소멸시효가 만료될 때까지 보관해야 합니다1.

급여 및 고용세 기록은 별도의 규정이 있습니다. 특정 연도의 4분기 신고 후 최소 4년 동안 유지해야 합니다1.

데이터 보안

납세자 데이터 보호는 법으로 요구되며 FTC(Federal Trade Commission)에 의해 시행됩니다1. 세무사는 문서화된 정보 보안 계획(WISP, Written Information Security Plan)을 가지고 있어야 합니다1.

 

WISP는 다음과 같은 내용을 포함해야 합니다:

  1. 프로그램을 조정할 담당자 지정
  2. 모든 위험과 현재 계획, 안전장치의 효과성 평가
  3. 안전장치 설계 및 구현
  4. 안전장치 모니터링 및 테스트
  5. 적절한 안전장치를 유지할 수 있는 서비스 제공업체 선택
  6. 필요에 따른 평가 및 조정

기본적인 보안 단계

  1. 피싱 이메일 인식 및 회피, 의심스러운 링크 클릭 금지
  2. 안티멀웨어 및 안티바이러스 소프트웨어 설치
  3. 8자 이상의 강력한 비밀번호 사용, 계정마다 다른 비밀번호 사용
  4. 민감한 파일과 이메일 암호화
  5. 민감한 데이터를 안전한 외부 소스에 백업
  6. 신고서 제출 전 최종 검토, 특히 은행 정보 확인
  7. 컴퓨터 폐기 시 하드 드라이브 완전 삭제 또는 파괴
  8. 납세자 데이터에 대한 접근을 필요한 개인으로 제한
  9. IRS 웹사이트에서 PTIN 및 EFIN 계정 확인
  10. 더 이상 고객이 아닌 경우 위임장(POA) 철회

데이터 유출 시 대응

데이터 유출이 발생한 경우, 즉시 IRS 이해관계자 연락 담당자(Stakeholder Liaison)에게 보고해야 합니다1. 필요한 경우 법 집행 기관에도 보고해야 할 수 있습니다1.

종이 문서 보안

종이 파일과 문서도 안전하게 보관해야 합니다1. 잠긴 방이나 파일 캐비닛에 보관하고, 화재나 홍수와 같은 손상으로부터 보호해야 합니다1. 문서 폐기 시에는 반드시 파쇄해야 합니다1.

전자 신고

전자 신고 제공자 신청

 

세무사로서 전자 신고 제공자가 되기 위해 신청할 수 있습니다1. 이를 위해서는 먼저 IRS.gov의 e-Services 계정이 필요합니다1. 온라인 신청서를 작성하고 전자 신고 발신자(Electronic Return Originator, ERO)를 선택해야 합니다1.

 

신청 과정에서 적합성 검사를 통과해야 하며, 이는 신용 조회, 세금 준수 확인, 범죄 경력 조회, 이전 전자 신고 요구사항 준수 여부 확인 등을 포함할 수 있습니다1. 이 과정은 최대 45일이 소요될 수 있습니다1.

 

승인되면 전자 신고 식별 번호(EFIN)가 발급됩니다1. 일반적으로 회사의 각 지점마다 하나의 EFIN을 받게 됩니다1.

전자 신고 의무

한 해에 11건 이상의 세금 신고서를 제출할 것으로 예상되는 세무사는 전자 신고를 해야 합니다1. 특별한 상황에서는 IRS에 면제를 신청할 수 있습니다(양식 8944 사용)1. 면제를 받은 경우, 종이로 제출하는 모든 신고서에 양식 8948(전자 신고를 하지 않는 이유 설명)을 포함해야 합니다1.

정보 신고서(W-2, 1099 등)의 경우, 10건 이상을 제출하는 경우 전자적으로 제출해야 합니다1. 이는 2023년부터 적용되는 새로운 규정으로, 이전 기준인 250건에서 크게 낮아졌습니다1.

전자 신고 발신자(ERO)의 역할

ERO는 승인된 IRS 전자 신고 제공자로, 세금 신고서의 전자 제출 과정을 시작합니다1. ERO는 다음 중 하나의 방법으로 전자 제출을 시작합니다:

  1. 전자적으로 신고서를 전송자(Transmitter)에게 보내 IRS로 전송
  2. 직접 IRS로 신고서 전송
  3. 중간 서비스 제공자(Intermediate Service Provider)에게 신고서 제공

실제로는 세금 소프트웨어를 통해 신고서를 제출하게 되며, 소프트웨어 제공업체가 전송자 또는 중간 서비스 제공자 역할을 하거나 이들과 계약을 맺어 IRS로 신고서를 전달합니다1.

ERO의 책임

ERO의 주요 책임은 다음과 같습니다:

  1. 전자 신고서의 적시 제출
  2. 필요한 보조 서류를 IRS에 제출
  3. 납세자에게 신고서 사본 제공
  4. 기록 보관 및 IRS 요청 시 제공
  5. 납세자와 제공자로부터만 신고서 수락
  6. 전자 신고 활동에서 발생한 수입 보고
  7. EFIN 보호

전자 신고 동의

전자 신고를 위해서는 납세자의 동의가 필요합니다1. 이는 양식 8879(전자 신고 서명 승인)에 서명함으로써 이루어집니다1. 이 양식은 신고서가 정확하다는 납세자의 선언이며, ERO에게 전자 신고 권한을 부여합니다1.

종이 문서 전송

전자 신고를 하더라도 일부 양식이나 문서는 종이로 제출해야 할 수 있습니다1. 이 경우 양식 8453을 사용하여 해당 문서를 제출합니다1. IRS가 전자 신고된 신고서를 수락했다는 확인을 받은 후 3영업일 이내에 양식 8453과 첨부 문서를 제출해야 합니다1.

전자 신고 거부 처리

전자 신고된 신고서가 거부될 수 있습니다1. 주로 정보 불일치로 인해 발생하며, 다음과 같은 경우가 있습니다:

  1. 사회보장번호나 생년월일 불일치
  2. 이미 같은 사회보장번호로 신고서가 제출된 경우
  3. 이미 다른 사람이 해당
